1.在父类定义通用的模板方法，具体细节方法由子类各自实现
2.例如盖房子，都需要打地基、砌墙搭、装修等步骤，这些方法就可以在父类的模板方法统一实现。但是具体的细节会根据不同的房子不一样，如别墅和草房需要的砌的墙和装修的工作不一样。
3.
优点： 1、封装不变部分，扩展可变部分。 2、提取公共代码，便于维护。 3、行为由父类控制，子类实现。

缺点：1、每一个不同的实现都需要一个子类来实现，导致类的个数增加，使得系统更加庞大。父类每增加一个方法所有的子类都要跟着修改，维护量大。

`4.以下是示例：

`public abstract class BuildTemplate {

/**
 * 打地基
 */
public abstract void layFoundation();

/**
 * 砌墙
 */
public abstract void masonryWall();

/**
 * 装修
 */
public abstract void renovation();

/**
 * 盖房子的模板方法
 */
public final void build(){

    /**
     * 1.先打地基 2.再砌墙 3.最后在装修
     */
    layFoundation();

    masonryWall();

    renovation();
}

}
`
/**

  • 盖别墅

  • /
    `public class BuildVilla extends BuildTemplate {
    @Override
    public void layFoundation() {

    System.out.println("别墅打地基");

    }

    @Override
    public void masonryWall() {

    System.out.println("别墅砌墙");

    }

    @Override
    public void renovation() {

    System.out.println("别墅装修");

    }
    }`

/**

  • 盖草房

  • /
    public class BuildThatchedCottage extends BuildTemplate{

    @Override
    public void layFoundation() {

    System.out.println("草房打地基");

    }

    @Override
    public void masonryWall() {

    System.out.println("草房砌墙");

    }

    @Override
    public void renovation() {

    System.out.println("草房装修");

    }
    }

public class Test {

public static void main(String[] args) {
    BuildTemplate buildTemplate = new BuildVilla();
    //现在我想盖一栋别墅 只要调用模板方法 build()就可
    buildTemplate.build();
    //想盖一栋草房也是一样
    buildTemplate = new BuildThatchedCottage();
    buildTemplate.build();
}

}
